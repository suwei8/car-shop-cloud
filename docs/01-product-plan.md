# 汽车行业服务管理系统产品方案初稿

## 1. 产品愿景

建设一套 SaaS 模式的汽车行业服务管理系统，面向汽修厂、汽车美容店、综合汽服门店和连锁门店，覆盖从客户获客、车辆建档、预约接待、维修施工、配件库存、会员储值、收银结算到经营分析的核心流程。

第一阶段目标不是做一个「百易云修克隆」，而是做一个结构清晰、可快速落地、从底层就支持多商户隔离、套餐订阅、功能开关，并在第一版同时交付 Web 管理后台和安卓 APP 的汽服经营 SaaS。

## 1.1 SaaS 模型原则

- 系统必须支持多商户，每个商户是独立租户。
- 所有业务数据必须按 `tenant_id` 隔离，门店级数据再按 `shop_id` 归属。
- 平台运营方需要独立后台，用于管理商户、套餐、到期、功能开关、用量、公告和系统配置。
- 商户侧需要独立后台，用于管理门店、员工、角色、客户、车辆、工单、库存和财务。
- 商户套餐决定可用模块、门店数量、员工数量、存储容量、短信额度、AI/高级报表等能力。
- 第一版必须支持多商户、多门店、多员工：平台把系统销售给商户，商户按套餐开通多个门店和员工账号。
- 任何跨商户查询、导出、统计都只能在平台运营后台以受控权限执行。

## 2. 目标客户

优先客户：

- 单店汽修厂：需要接车、开单、派工、库存、收银。
- 汽车美容/洗美店：需要快速开单、套餐卡、储值卡、核销、预约。
- 中小型综合汽服门店：同时做机修、保养、洗美、轮胎、基础检查。
- 多门店汽服公司：需要总部查看门店经营、统一客户资产、跨店会员或套餐。

后续客户：

- 配件仓/云仓：需要采购、调拨、库存协同。

## 3. 角色设计

| 角色 | 核心目标 | 主要功能 |
| --- | --- | --- |
| 老板/经营者 | 看经营、控风险、看利润 | 工作台、经营报表、财务统计、库存预警、客户流失提醒 |
| 店长 | 管流程、管人员、管待办 | 今日接车、在修车辆、派工、异常工单、回访 |
| 前台/接待 | 快速接车、报价、收款 | 客户车辆检索、预约、接车、报价、工单、结算 |
| 维修技师 | 查看任务、记录施工 | 我的派工、开工/完工、施工备注、图片、用料申请 |
| 仓管 | 管配件库存和领料 | 入库、库存、领料出库、退料、盘点、预警 |
| 财务 | 审核收支和结算 | 收款、欠款、应收、对账、封账、报表 |
| 商户管理员 | 管账号和配置 | 员工、角色权限、门店、系统参数、打印模板 |
| 平台运营 | 管 SaaS 商户 | 商户开通、套餐、功能开关、到期续费、平台公告 |

## 4. MVP 范围

### 必做模块

1. 工作台
   - 今日到店、在修工单、待结算、待回访、库存预警。
   - 按角色显示不同待办。

2. 客户与车辆
   - 客户档案、车辆档案、联系人、手机号、车牌、VIN、品牌车型、里程。
   - 一个客户多辆车，一辆车可记录使用人和车主。
   - 维修历史、消费历史、套餐/储值余额。

3. 预约与接待
   - 预约登记、预约确认、到店转接车。
   - 快速车牌/手机号搜索。
   - 接车单记录进店时间、里程、油量、故障描述、随车物品、照片。

4. 维修工单
   - 工单创建、服务项目、配件项目、增值服务、优惠。
   - 工单状态：草稿、待派工、施工中、待结算、已结算、已交车、作废。
   - 支持维修单与洗美快单，底层同一工单模型，通过业务类型和页面体验区分。
   - 健康检查一期做轻量版，支持检查项目、图片、备注和检查结果。

5. 派工与施工
   - 按项目派工到技师/班组。
   - 技师开工、暂停、完工、备注、上传图片。
   - 车间看板展示车牌、项目、技师、预计完工时间、状态。

6. 配件库存
   - 配件档案、供应商、仓库、库存浏览。
   - 采购入库、维修出库、退料、库存调整、盘点、库存预警。
   - 第一期默认采用「工单确认施工/出库时直接扣库存」，不等结算，避免已用料但未结算导致库存不准。
   - 领料单/仓库审核流程作为二期扩展，不阻塞第一版上线。

7. 储值卡
   - 第一期必做储值卡：开卡、充值、赠送金额、余额、消费扣款、退款/作废、停用、流水。
   - 储值卡需要支持绑定客户，可扩展限制适用车辆、适用门店、有效期。
   - 工单结算时支持使用储值余额抵扣，并写入储值流水和结算明细。
   - 储值金额建议拆分本金余额和赠送余额，便于退款、核销和财务统计。

8. 套餐卡
   - 第一期必做套餐卡：售卡、项目次数、剩余次数、有效期、适用车辆限制、适用门店限制、核销、退款/作废、流水。
   - 套餐卡用于按服务项目或项目包进行次数核销，和储值卡的金额抵扣分开处理。
   - 工单结算时支持选择可用套餐项目抵扣服务项目，并写入套餐核销流水和结算明细。

9. 收银结算
   - 工单应收、优惠、实收、欠款。
   - 多支付方式组合收款，包含现金、微信、支付宝、储值卡、挂账。
   - 第一期不接微信/支付宝真实支付，只记录收款方式、金额和流水号。
   - 结算单、收款记录、退款记录。

10. 报表
   - 营业日报、工单统计、项目收入、配件收入、技师产值、客户新增、储值余额、库存预警。

11. 系统管理
   - 门店、员工、角色、权限、基础字典、业务类型、项目分类、编码规则、A4 打印模板。

12. 平台运营后台
   - 商户管理、套餐版本、功能开关、到期时间、门店/员工额度、平台公告、操作审计。

13. 安卓 APP
   - 面向门店员工现场作业：接待开单、拍照、派工任务、施工完工、移动收款记录、看板提醒。
   - 一期优先覆盖接待、技师、店长，财务和系统配置仍以 Web 后台为主。

### 暂缓模块

- 拼团、抽奖、复杂卡券营销。
- 企微深度集成、短信营销。
- 云仓和商城采购。
- 工资、社保、复杂绩效。
- EPC/保养手册自建数据库。
- 保险业务。
- 微信小程序。
- 真实微信/支付宝支付。
- 历史数据导入迁移。
- 复杂多级总部管控与跨公司财务。

## 6. 关键产品建议

### 6.1 不要把所有业务都做成单据列表

汽服门店现场节奏快，前台和车间需要看板：

- 接待看板：今日预约、等待接待、待报价、待结算。
- 车间看板：车牌、项目、技师、工位、预计完工、超时提醒。
- 老板看板：营业额、到店数、在修数、毛利、欠款、库存预警。

### 6.2 工单要支持多业务类型

同一个门店可能做保养、机修、钣喷、洗美、轮胎等业务。第一期明确支持维修和洗美两条业务：

- 维修工单：信息更完整，适合机修、保养、钣喷等业务。
- 洗美快单：开单路径更短，适合洗车、美容、精品加装等高频业务。
- 底层使用统一工单模型，业务类型做可配置字典，避免每类业务单独开发一套流程。
- 健康检查做轻量版，作为工单的检查记录和图片附件；保险业务暂不纳入当前规划。

### 6.3 第一期采用直接扣库存

用户已确认第一期要做库存，且采用工单确认施工/出库时直接扣库存。产品含义：

- 配件档案、仓库、库存余额、入库、出库、退料、调整、盘点、预警一期必须具备。
- 工单配件明细与库存打通，工单确认施工或配件确认出库时直接生成库存流水并扣减库存。
- 不等结算再扣库存，避免在修未结算车辆占用配件但系统库存仍虚高。
- 默认不做复杂领料审核，不要求仓管二次确认。
- 必须记录库存流水，保证后续可以追溯每一次扣减来自哪张工单、哪个配件明细、哪个员工。
- 二期再扩展标准领料出库、领料退货、仓库审核、跨仓调拨等流程。

### 6.4 储值卡与套餐卡一期必做

用户已确认储值卡和套餐卡第一期都要做。二者都属于客户资产，但处理对象不同：储值卡管金额，套餐卡管项目次数。

储值卡要求：

- 开卡、充值、消费、退款、作废、停用都必须形成流水。
- 储值余额应区分本金和赠送金额，避免退款和收入统计混乱。
- 工单结算使用储值卡时，要同时更新结算单、支付记录、储值流水。
- 支持按客户绑定，预留按车辆、门店、有效期限制的能力。
- 安卓 APP 和 Web 后台可发起储值卡收款抵扣，并查看储值余额和消费记录。

套餐卡要求：

- 售卡、核销、退款、作废、停用都必须形成流水。
- 套餐卡至少支持按服务项目配置次数，例如洗车 10 次、保养 3 次。
- 必须支持有效期、适用车辆、适用门店限制；不符合限制时禁止在工单中核销。
- 工单结算时，服务项目可使用套餐卡次数抵扣，抵扣后减少剩余次数。
- 安卓 APP 和 Web 后台需要展示套餐项目、剩余次数、有效期和核销记录。

### 6.5 权限要到按钮和数据范围

至少支持：

- 菜单权限。
- 按钮权限：新增、编辑、删除、审核、反结算、作废、导出、打印。
- 数据范围：本人、所在门店、全部门店。
- 敏感字段脱敏：手机号、车牌、金额可按权限控制。

## 7. 推荐端设计

| 端 | 用户 | 建议技术形态 |
| --- | --- | --- |
| 平台运营后台 | SaaS 平台运营、客服、管理员 | Web，独立权限空间 |
| 商户 Web 管理后台 | 老板、店长、前台、仓管、财务 | Web，信息密度高，承载完整管理能力 |
| 安卓 APP | 接待、技师、店长、老板 | uni-app/Flutter/React Native，优先复用 API |
| 大屏看板 | 前台、车间 | Web 大屏，轮播刷新 |
| 后续开放 API | 第三方、公众号、合作平台 | REST/开放平台 |

### 7.1 端职责边界

Web 管理后台：

- 平台运营后台和商户后台。
- 完整的配置、财务、库存、报表、权限、系统参数。
- 复杂表格、审核、打印模板。

安卓 APP：

- 员工端优先：接待拍照、车辆检查、开单、派工、施工任务、完工、移动收款记录。
- 支持离店/车间现场操作，减少对电脑的依赖。
- 需要和 Web 后台共享同一套权限、工单状态和文件上传能力。
