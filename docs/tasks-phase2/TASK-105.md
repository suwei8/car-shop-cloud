# TASK-105：新租户初始化（权限 seed / 预置服务项目库 / 开箱即用）

> **优先级**：P1
> **状态**：待派发
> **依赖**：无
> **可并行**：TASK-101~104

## 1. 任务目标

让新商户"注册后 10 分钟能开出第一张单"。当前新建租户后没有默认角色权限、没有门店、没有仓库、没有任何服务项目，老板面对一个空系统无从下手——这是试用转化的最大杀手。

本任务实现：**租户初始化器**（创建租户时自动执行）+ **权限全量 seed 脚本**（修复已知的"权限初始化缺失"问题）。

## 2. 涉及文件

### 新建文件
- `apps/api/src/platform/tenant/tenant-initializer.service.ts` — 租户初始化器
- `apps/api/prisma/seed-data/permissions.ts` — 权限定义清单（全量）
- `apps/api/prisma/seed-data/service-items.ts` — 预置服务项目库数据
- `apps/api/prisma/seed-data/dictionaries.ts` — 预置字典数据

### 修改文件
- `apps/api/prisma/seed.ts` — 引入权限全量 seed（幂等 upsert）
- `apps/api/src/platform/tenant/tenant.service.ts` — 创建租户后调用初始化器
- `apps/api/src/platform/tenant/tenant.module.ts`

## 3. 详细要求

### 3.1 权限全量 seed（修复缺口）

- 梳理代码中所有 `@RequirePermissions()` 用到的权限码（用 grep 全量扫描 `apps/api/src`），整理为 `permissions.ts` 数据文件（code、name、module）
- `seed.ts` 中按 code upsert，**幂等可重复执行**
- 若发现 controller 中使用了 seed 中没有的权限码、或 seed 中有代码未使用的权限码，在回执中列出差异清单

### 3.2 租户初始化器（事务内执行）

创建租户时自动完成：

1. **默认门店**：用租户名创建 1 个 Shop（总店）
2. **默认仓库**：为该门店创建 `isDefault: true` 的 Warehouse
3. **内置角色 + 权限绑定**（使用 `@car/shared` 的 BUILT_IN_ROLES）：
   - `tenant_admin` → 全部 tenant: 权限
   - `shop_manager` → 除角色/员工管理外的全部
   - `service_advisor` → 客户/车辆/预约/工单/结算查看与创建
   - `technician` → 工单查看、派工任务操作
   - `cashier` → 结算、储值卡、套餐卡
   - 角色 `isBuiltIn: true`，商户端不可删除（role.service 删除时校验，如已有此逻辑则确认即可）
4. **管理员账号**：为创建时传入的联系人手机号创建 User，绑定 `tenant_admin` 角色，初始密码随机生成并在创建接口响应中**仅此一次**返回
5. **预置服务项目**（约 30 条，常见汽服项目，分类建议：洗美/保养/维修/轮胎）：

   示例（价格仅作默认值，商户可改）：洗车-标准洗车 ¥30、精洗 ¥80、打蜡 ¥200、内饰清洗 ¥150；保养-机油机滤小保养 ¥0（工时费另计）、空气滤芯更换 ¥30、空调滤芯更换 ¥40、刹车油更换 ¥120、防冻液更换 ¥100；维修-故障诊断 ¥50、刹车片更换（工时）¥80、电瓶更换（工时）¥30；轮胎-补胎 ¥40、换胎（只）¥20、四轮定位 ¥150、轮胎换位 ¥40 等，补齐至约 30 条
6. **预置字典**：业务类型（维修/保养/洗美/轮胎/其他）、支付方式、常见车辆颜色等现有字典模块支持的类型

### 3.3 已存在租户的补救

提供一次性脚本 `apps/api/prisma/init-existing-tenants.ts`（ts-node 运行）：对已存在但缺少初始化数据的租户补齐（幂等：已有同名角色/项目则跳过）。

### 3.4 单元测试

- 初始化器测试：mock prisma，验证一次创建产生门店/仓库/角色/项目的调用完整性、事务包裹
- 重复初始化幂等性测试

## 4. 验收标准

- [ ] `pnpm db:seed` 幂等可重复执行，权限全量入库
- [ ] 新建租户后自动拥有：1 门店、1 默认仓库、5 内置角色（权限绑定正确）、管理员账号、约 30 条服务项目、基础字典
- [ ] 管理员初始密码只在创建响应中返回一次，数据库存 bcrypt 哈希
- [ ] 用新租户管理员登录 → 直接创建一张含预置服务项目的工单可走通（回执需附验证过程）
- [ ] 权限码差异清单已在回执列出（或确认无差异）
- [ ] 一次性补救脚本可对存量租户执行且幂等
- [ ] 新增单元测试通过；`nest build` 通过

## 5. 注意事项

- 初始化必须在事务中，失败时整体回滚（避免半初始化租户）
- 服务项目价格用 Decimal，预置数据中金额写字符串如 `"30.00"`
- 不要修改现有租户业务模块的查询逻辑
- TASK-106（自助注册）将复用本初始化器，注意把它做成独立可注入的 service
