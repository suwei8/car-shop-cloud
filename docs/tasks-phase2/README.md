## 工作流

```
PM 编写任务书 → 用户复制派发词给执行 Agent → Agent 按任务书执行
    → Agent 将回执填入任务书第 6 节 → 用户通知 PM 审核
    → 审核通过：关闭任务 / 未通过：PM 生成整改提示词 → 用户交给 Agent 整改 → 循环
```

## 执行 Agent 通用要求（每个任务都适用）

1. 严格按任务书范围执行，**不做范围外的"顺手优化"**
2. 遵守项目硬约束（见根目录 AGENTS.md）：租户隔离、金额 Decimal、资金类操作写流水
3. 完成后必须将回执**追加写入任务书第 6 节回执区域**（填写表格，不得改动任务书其他内容，尤其不得改动第 7 节审核区域）
4. 验收标准中标注"必填"的验证项必须实际执行并附过程，不得只声明"已完成"
5. 提交前确保 `nest build` 通过；涉及 web 的任务确保 `vue-tsc` 通过；新增测试全部通过
6. **若任务书的约束与实际环境发生冲突（如某约束导致验收标准无法达成），禁止擅自突破约束**；应在不违反约束的前提下寻找替代方案，实在无解则停止该项并在回执"已知限制"中如实说明，由审核人决定
7. 回执中的每一项"通过"声明都必须是实际执行过的结果，禁止声称未实际验证的结论
