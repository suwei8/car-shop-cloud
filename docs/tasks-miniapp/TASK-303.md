# TASK-303 — 商户端小程序核心 5 屏

> 优先级：P0　依赖：TASK-302（建议 TASK-304 先行或并行以补齐欠款/质保数据）

## 1. 背景

第一版小程序范围锁定为：**注册 + 核心 5 屏 + 套餐支付**。本任务实现核心 5 屏。现有移动端已有部分页面可复用：`pages/index`(首页)、`pages/workorder/*`(工单)、`pages/search`(查客户)、`pages/member/card`(会员卡)、`pages/profile`(我的)。

## 2. 五屏要求

### 2.1 首页（老板看账）
展示：今日收入、今日工单数、**客户欠款总额**、低库存预警数、待办提醒（保养到期/客户流失）；
快捷入口：开维修单、开洗车单、查客户、收欠款、会员充值。
> 欠款字段依赖 TASK-304（Dashboard 欠款统计）；若 304 未完成，先预留 UI 并标注。

### 2.2 开单
- 车牌/手机号检索老客户，自动带出车辆；
- 选服务项目 + 配件（配件可查库存）；金额自动计算；
- 简易模式「一键完工」；
- 复用/完善 `pages/workorder/create.vue`、`detail.vue`。

### 2.3 客户
- 车牌/手机号/VIN 搜索；客户车辆档案、维修/消费历史；储值卡/次卡余额；
- **质保查询**：展示该车已更换配件的质保截止与是否在保（依赖 TASK-304 质保追溯）；
- 复用/完善 `pages/search/search.vue`。

### 2.4 配件库存（新增页）
- 新增 `pages/stock/list.vue` 并注册 `pages.json`；
- 配件搜索、当前库存、低库存高亮、质保月数、供应商及电话；
- 简单入库（选配件、数量、进价、供应商、质保）——若入库较重可一期只做「查询 + 低库存」，入库留 Web，**在回执说明取舍**。

### 2.5 我的店
- 门店信息、套餐与试用状态、续费入口（续费/支付对接 TASK-305）；
- **员工管理（按需入口）**：邀请二维码为主 + 手机号添加为辅；角色简化为「老板/员工」；员工管理非注册必经，老板需要时点击；
- 跳转 Web 后台引导（打印/报表/导入等重操作）。

## 3. 约束

- 全部为商户端，无车主端；
- 面向小店老板，界面简单、操作短；
- 简易模式下隐藏派工、多仓库、复杂权限等；
- 储值卡/次卡按「简化原则」展示（开卡/充值/赠送/消费/查余额/退卡），不做会员等级；
- 保证 `build:mp-weixin` 可编译。
