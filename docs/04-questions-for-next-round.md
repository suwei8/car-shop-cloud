# 下一轮需求确认问题

这些问题用于把产品范围从「参考百易云修」收敛到我们自己的第一版。

## 1. 目标客户

1. 第一版采用 SaaS 多商户、多门店、多员工模式；商户按套餐添加门店和员工。
2. 还需要确认套餐分级：基础版、专业版、连锁版分别允许多少门店和员工？
3. 目标首批客户更偏汽修厂、洗美店、综合汽服店，还是连锁门店？

## 2. 业务范围

4. 第一版已确定做库存，不需要历史数据导入。
5. 库存扣减时点已确定：工单确认施工/出库时扣，不等结算。
6. 第一版已确定支持维修工单 + 洗美快单两种业务体验。
7. 健康检查一期做轻量版。
8. 保险业务暂不纳入当前产品规划。

## 3. 会员营销

9. 储值卡已确定一期必做；充值是否需要支持赠送金额，例如充 1000 送 100？
10. 套餐卡已确定一期必做，并且必须限制适用车辆、适用门店、有效期。
11. 是否需要优惠券、短信营销、企微回访这类营销功能？

## 4. 财务结算

12. 第一期已确定不接真实微信/支付宝支付，只记录收款方式、金额和流水号。
13. 是否允许挂账和欠款？
14. 是否需要财务审核、反结算、封账？
15. 是否需要应收应付，还是只做工单收款？

## 5. 端与设备

16. 第一版已调整为 Web 管理后台 + 安卓 APP，不做微信小程序。
17. 安卓 APP 首版主要给哪些角色使用：接待、技师、店长、老板？
18. 门店是否需要大屏看板？
19. A4 工单和 A4 结算单一期必做；是否还要热敏小票或标签打印？

## 6. 数据与集成

20. 已确认没有历史数据需要导入。
21. 是否需要 VIN 解析、车型库、EPC、保养手册等第三方数据？
22. 微信小程序不进第一版，如后续需要可作为二期独立规划。
23. 是否需要对接现有财务软件或进销存软件？

## 7. 部署与商业化

24. SaaS 套餐准备如何分级：基础版、专业版、连锁版，还是先不分级？
25. 是否需要试用期、商户到期、续费提醒、到期冻结？
26. 是否需要支持私有化部署？
27. 是否已有品牌名称、Logo、域名？

## 8. 我建议的默认答案

如果你希望尽快落地，我建议第一版默认：

- 多商户 SaaS 架构作为硬前提，一期就实现租户隔离。
- SaaS 第一版支持多商户、多门店、多员工，商户按套餐限制门店和员工数量。
- 第一版包含 Web 管理后台 + 安卓 APP。
- 安卓 APP 面向门店员工现场作业。
- 一期必做客户车辆、预约接待、维修工单、派工、库存管理、结算、储值卡。
- 套餐卡第一期也要做，按服务项目次数核销。
- 套餐卡必须校验适用车辆、适用门店和有效期。
- 库存第一期采用工单确认施工/出库时直接扣库存，不做仓库领料审核流程。
- 一期支持维修工单 + 洗美快单，轻量健康检查；保险业务暂不纳入当前规划。
- A4 工单和 A4 结算单一期必做。
- 一期没有历史数据导入。
- 暂不接真实支付，只记录支付方式和流水号。
- 暂不做复杂营销、工资社保、云仓、EPC 自建库。
