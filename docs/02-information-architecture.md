# 信息架构与数据模型初稿

## 1. 后台导航建议

系统采用 SaaS 模型，信息架构拆成两套后台：平台运营后台和商户管理后台。平台后台服务 SaaS 运营方，商户后台服务汽服门店。建议商户后台一级导航保持在 8 到 10 个，避免百易云修 200 多菜单直接暴露造成学习成本过高。

## 1.1 平台运营后台

1. 平台概览
2. 商户管理
3. 套餐版本
4. 功能开关
5. 订单与续费
6. 用量监控
7. 平台公告
8. 系统审计
9. 平台配置

平台后台关键能力：

- 开通、停用、冻结、恢复商户。
- 配置商户套餐、到期时间、门店数量、员工数量、存储容量。
- 控制模块能力，例如库存、储值卡、套餐卡、报表、移动端、短信、AI。
- 查看跨商户用量和异常，但默认不能查看商户业务敏感数据。
- 发布平台公告和系统升级通知。

## 1.2 商户管理后台

## 4. 工单明细设计

建议使用统一工单明细模型：

| 字段 | 说明 |
| --- | --- |
| itemType | service、part、value_added、suggestion |
| refId | 服务项目、配件、增值服务或空 |
| name | 开单名称，允许快录 |
| quantity | 数量 |
| unitPrice | 单价 |
| discountRate | 折扣率 |
| amount | 金额 |
| costPrice | 成本 |
| technicianId | 技师 |
| teamId | 班组 |
| status | 待施工、施工中、完成、取消 |
| stockPolicy | 第一期默认为直接扣库存，二期扩展领料扣 |

这样可以支持百易云修中出现的服务项目、配件、增值服务、建议项目，也方便后续扩展外包项目。

## 5. 权限模型

权限分为平台侧和商户侧。

平台侧权限：

- 平台超级管理员：管理平台配置、套餐、商户。
- 平台运营：开通商户、处理续费、发布公告。
- 平台客服：查看商户基础信息和使用状态，不默认查看业务敏感数据。

商户侧权限分三层：

- 菜单权限：是否可进入模块。
- 操作权限：新增、编辑、删除、审核、反审核、作废、结算、反结算、导出、打印。
- 数据权限：本人、本门店、多门店、全部。

敏感能力建议单独控制：

- 查看完整手机号
- 查看完整车牌/VIN
- 修改结算金额
- 反结算
- 删除单据
- 导出客户资料
- 查看成本和毛利

## 5.1 第一期库存模型

第一期库存采用「工单确认施工/出库时直接扣库存」：

- 配件入库增加库存。
- 工单确认施工或配件确认出库后直接扣减库存。
- 不等工单结算再扣减库存。
- 退料或工单取消时按规则回补库存。
- 库存调整和盘点必须生成库存流水。
- 所有库存变化必须记录来源业务对象，便于审计。

一期必须支持的库存单据：

- 采购入库
- 工单直接出库
- 工单退料
- 库存调整
- 库存盘点

二期再扩展：

- 领料申请
- 仓管审核
- 领料出库
- 领料退货
- 跨仓调拨
- 云仓协同

## 5.2 第一期储值卡模型

第一期储值卡必须支持：

- 开卡
- 充值
- 赠送金额
- 工单消费抵扣
- 退款
- 作废
- 停用/启用
- 储值流水
- 安卓 APP 余额和消费记录查询

储值卡余额建议拆成：

- 本金余额：客户实际支付形成的余额。
- 赠送余额：营销赠送形成的余额。
- 总余额：本金余额 + 赠送余额。

工单结算使用储值卡时，需要同时写入：

- 结算单支付明细
- 储值卡余额变更
- 储值流水
- 操作日志

## 5.3 第一期套餐卡模型

第一期套餐卡必须支持：

- 售卡
- 项目次数配置
- 有效期
- 适用车辆
- 适用门店
- 工单项目核销
- 退款
- 作废
- 停用/启用
- 套餐流水
- 安卓 APP 套餐剩余次数和核销记录查询

套餐卡核销限制必须在后端校验：

- 车辆限制：工单车辆必须在套餐卡适用车辆范围内。
- 门店限制：工单门店必须在套餐卡适用门店范围内。
- 有效期限制：核销时间必须在套餐卡有效期内。
- 次数限制：对应套餐项目剩余次数必须大于 0。
- 状态限制：套餐卡必须处于正常状态。

套餐卡与储值卡的结算差异：

- 储值卡抵扣金额，影响应收/实收。
- 套餐卡核销次数，抵扣具体服务项目。
- 同一工单可以同时使用套餐卡和储值卡，但必须分别写入套餐核销流水和储值流水。

## 5.4 第一期工单业务边界

第一期支持两个业务入口：

- 维修工单：面向维修、保养、机修、钣喷等标准服务流程。
- 洗美快单：面向洗车、美容、精品等高频低复杂度服务。

底层统一使用 `WorkOrder` 和 `WorkOrderItem`，通过业务类型、是否快单、页面表单配置区分体验。

健康检查一期做轻量版：

- 检查项
- 检查结果
- 图片
- 备注
- 关联工单和车辆

保险业务暂不纳入当前产品规划。

## 6. 系统参数

门店级参数建议至少包括：

- 库存扣减模式：第一期固定为工单确认施工/出库时直接扣库存；二期再开放领料扣、不启用等配置。
- 工单是否必须派工。
- 工单是否必须质检。
- 结算是否需要审核。
- 是否允许负库存。
- 是否允许修改已结算工单。
- 预约过期时间。
- 看板刷新时间。
- 编码规则前缀。
- 打印模板。

商户级参数建议包括：

- 商户状态：试用、正常、到期、冻结、停用。
- 套餐版本。
- 到期时间。
- 门店数量上限。
- 员工数量上限。
- 存储容量上限。
- 短信额度。
- 可用模块清单。
- 是否启用多门店，第一版必须支持，具体数量由商户套餐限制。

## 7. 编码规则建议

参考百易云修编码形态，可采用门店号 + 类型 + 年月日 + 流水：

- 工单：`A01-WX-20260527-0001`
- 洗美单：`A01-XM-20260527-0001`
- 客户：`A01-C-20260527-0001`
- 车辆：`A01-V-20260527-0001`
- 采购入库：`A01-RK-20260527-0001`
- 工单出库：`A01-CK-20260527-0001`
- 储值卡：`A01-CZ-20260527-0001`
- 套餐卡：`A01-TC-20260527-0001`

实际展示可更短，但数据库中要保证租户内唯一。

推荐按业务域拆分：

- `/api/auth`
- `/api/shops`
- `/api/customers`
- `/api/vehicles`
- `/api/appointments`
- `/api/receptions`
- `/api/work-orders`
- `/api/dispatch-tasks`
- `/api/parts`
- `/api/inventory`
- `/api/member-cards`
- `/api/settlements`
- `/api/reports`
- `/api/settings`

列表接口统一支持：

- 分页：page、pageSize
- 排序：sort
- 筛选：filter
- 字段投影：fields，可选
- 导出：异步任务，避免大查询阻塞

## 9. 与百易云修差异化方向

1. 降低菜单复杂度：默认简洁导航，高级模块按门店启用。
2. 强化移动端体验：技师任务、接待拍照、移动收款优先。
3. 强化数据可视化：老板工作台、现金流、毛利、复购提醒。
4. 强化流程可配置：一期库存直接扣减，后续再开放领料、结算审核、派工/质检开关。
5. 强化导入迁移：用户已确认一期没有历史数据需要导入，导入迁移放二期。
