## 1. 背景

当前项目经历了多轮断续开发，文档中存在以下风险：

- 部分任务已经完成但 README 或任务正文仍显示待执行；
- 部分历史任务方向与当前产品定位不一致；
- 部分任务文件已删除或迁移，但索引仍可能残留引用；
- 当前产品方向已修正为小型汽车服务门店内部使用，不做车主端，不以大型汽修厂高端系统为主要对象。

当前权威产品方向以 `docs/09-current-product-direction.md` 为准。

## 3. 状态分类规则

请把任务状态统一归类为以下之一：

- `✅ 已关闭`：代码或文档证据显示任务已经完成，并且不需要继续执行。
- `🟡 部分完成`：已有部分实现，但仍有明确剩余项。
- `待执行`：当前方向仍需要，但项目中缺少完成证据。
- `已废弃`：与当前产品方向冲突，或已被新任务替代。
- `历史参考`：保留作为背景资料，不作为待执行任务。

判定必须写出证据，例如：

- 相关文件已存在；
- 相关测试已存在；
- README 已被新文档替代；
- 当前方向明确排除车主端；
- 任务依赖的代码模块不存在或已被其他实现替代。
