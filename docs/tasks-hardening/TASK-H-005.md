# TASK-H-005 — 灰度验收链路

> 优先级：P1
> 状态：✅ 已关闭
> 前置决策：`TASK-H-004 真实短信上线验证` 已暂缓，生产使用前再对接真实短信供应商。
> 产品边界：面向小型汽车服务门店的内部经营工具，优先微信小程序；暂不做车主端。

## 1. 背景

当前项目已经完成多轮功能开发和若干技术收口，但还缺少一条面向灰度上线的、可重复执行的业务验收链路。

这条链路不是完整自动化 E2E 测试框架，也不是生产压测；它的目标是让我们在交付给小型门店试用前，能用固定清单和尽量轻量的脚本确认核心业务闭环没有断点。

由于真实短信上线验证暂缓，本任务不得接入真实短信供应商，也不得写入真实短信密钥。注册环节可以采用已有开发验证码、已有测试账号、测试租户或人工准备的验证码。

## 2. 目标

建立一套灰度验收材料，覆盖小门店日常经营最关键路径：

1. 门店老板或员工能完成测试租户注册/登录。
2. 能建立客户和车辆档案。
3. 能建立商品/配件并完成入库，或确认测试商品有可用库存。
4. 能创建工单，添加服务项目和配件。
5. 工单完工或结算时能正确形成库存扣减记录。
6. 能完成线下收款记录，不依赖真实微信/支付宝支付。
7. 能查询工单、库存流水、收款状态和质保信息。
8. SaaS 订阅续费链路只能使用 mock/sandbox/测试环境，不触发真实扣款。

## 6. 可选 smoke 脚本要求

如果实现 `gray-smoke.ts`，必须满足：

1. 默认只读或直接退出，不得默认写数据。
2. 涉及创建客户、车辆、库存、工单、结算等写操作时，必须要求：

   ```bash
   SMOKE_ALLOW_WRITE=1
   ```

3. 必须要求显式传入 API 地址，例如：

   ```bash
   API_BASE_URL=http://127.0.0.1:3000/api
   ```

4. 不得内置真实手机号、真实验证码、真实密钥。
5. 失败时输出具体失败步骤和 HTTP 状态/响应摘要。
6. 脚本要能被 `pnpm --filter @car/api smoke:gray` 调用。
7. 如果受限于当前接口或数据模型无法实现完整链路，请实现可稳定执行的部分，并把剩余步骤留在人工清单中。

建议环境变量：


#### 发现的阻塞点或风险

1. **测试账号不存在**：当前数据库中没有 `18800000001` 的测试账号。Smoke 脚本无法完成完整链路。需人工先执行注册或手动插入测试数据。
2. **注册依赖短信验证码**：即使 `SMS_PROVIDER=mock`，仍需先调用 `send-code` 获取验证码后才能注册。Smoke 脚本当前未自动完成注册流程（因注册会创建新租户，可能干扰现有数据）。
3. **部分接口依赖前置数据**：结算、出库等操作依赖工单和库存数据，需要按正确顺序执行。Smoke 脚本已内置正确的执行顺序。
4. **无真实短信/支付**：本轮不接入真实供应商，验收仅覆盖 mock 模式。真实环境需另行验证。
5. **无前端验收**：本清单和脚本仅覆盖 API 层面。前端 UI 交互、微信小程序流程需另行验收。
