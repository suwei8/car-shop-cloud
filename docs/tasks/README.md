## 状态说明

- **待派发**：任务书已编写，尚未交给 agent 执行
- **执行中**：agent 正在执行
- **回执已提交**：agent 已完成并填写回执，等待架构师审核
- **审核通过**：架构师确认达标
- **需整改**：架构师审核未通过，已给出整改提示词
- **已关闭**：任务完成并归档

## 工作流

```
架构师编写任务书
    ↓
用户复制派发词给 Agent
    ↓
Agent 执行开发任务
    ↓
Agent 填写回执（修改了哪些文件、测试方法、已知限制）
    ↓
用户将回执反馈给架构师
    ↓
架构师审核 → 通过 → 关闭任务
              ↓
           未通过 → 生成整改提示词 → 用户交给 Agent 整改 → 循环
```
