# 车店云管家开发交接说明

本目录是「车店云管家」第一版开发的产品与架构交接包。远程开发 agent 开始实现前，请先按顺序阅读本文件列出的文档。

## 1. 当前最终决策

- 产品名暂定：车店云管家。
- 商业模式：SaaS 年费套餐 + 增值服务。
- 第一版端形态：Web 管理后台 + 安卓 APP。
- 第一版不做微信小程序；小程序后续按市场需要独立规划。
- 第一版不做保险业务。
- 第一版不接真实微信/支付宝支付，只记录收款方式、金额和流水号。
- 第一版不做历史数据导入。
- 第一版必须支持多商户、多门店、多员工，商户按套餐限制门店数和员工数。
- 第一版支持维修工单 + 洗美快单。
- 第一版支持轻量健康检查。
- 第一版支持库存，扣减时点为工单确认施工/出库时直接扣库存，不等结算。
- 第一版支持储值卡和套餐卡。
- 套餐卡必须限制适用车辆、适用门店、有效期。
- 第一版支持 A4 工单和 A4 结算单打印。

## 3. 建议第一阶段工程任务

Sprint 0：

- 初始化后端、Web 前端、安卓 APP 工程。
- 建立租户、商户、门店、员工、角色、权限、套餐、功能开关底座。
- 建立数据库迁移和种子数据。
- 建立登录鉴权、租户隔离、文件上传、审计日志。

Sprint 1：

- 客户档案、车辆档案。
- 预约接待。
- 维修工单和洗美快单基础流程。
- 安卓 APP 员工登录、快速查车、接待开单。

Sprint 2：

- 派工、施工、轻量健康检查。
- 库存入库、库存余额、工单确认施工/出库扣库存。
- A4 工单打印。

Sprint 3：

- 结算、收款记录、储值卡、套餐卡、A4 结算单。
- 套餐卡核销限制：车辆、门店、有效期、剩余次数、状态。

Sprint 4：

- 工作台、报表、库存预警、技师产值。
- 安卓 APP 施工拍照、开工/完工、移动收款记录。

## 4. 开发硬约束

- 所有业务表必须包含 `tenant_id`。
- 门店级业务数据必须包含 `shop_id`。
- 后端必须强制租户隔离，不能依赖前端传入 `tenant_id`。
- 套餐和功能开关必须后端校验。
- 金额不能使用 float。
- 库存、结算、储值卡、套餐卡必须写流水和审计日志。
- 已结算工单不能直接修改核心金额，应通过反结算或调整流程处理。
