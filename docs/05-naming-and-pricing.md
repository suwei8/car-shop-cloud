# 产品命名与收费方案建议

## 1. 命名方向

产品面向汽车修理厂、洗车店、综合汽服门店，以 SaaS 年费方式销售。名字需要满足：

- 一听就知道服务汽修/汽服行业。
- 比「ERP」更容易被中小门店老板理解。
- 适合上架安卓应用商店。
- 后续可扩展到洗美、会员、库存、门店经营，不局限于“修”。

正式定名前必须再做：

- 商标近似查询。
- 公司/产品名重名查询。
- 域名查询。
- 安卓应用商店重名查询。

## 2. 推荐主名称

### 首选：车店云管家

推荐理由：

- “车店”覆盖汽修厂、洗车店、美容店，比“汽修”更宽。
- “云管家”容易让老板理解为门店管理系统。
- 亲和、低门槛，适合年费 SaaS 销售。
- 安卓 APP 名称可用「车店云管家」或「车店云管家商户端」。

可用标语：

- 汽服门店经营管理系统
- 修车洗美一套管
- 汽服门店的云端管家

### 备选 1：汽服云掌柜

推荐理由：

- “掌柜”有经营意味，适合老板视角。
- 覆盖汽修、洗美、轮胎、保养等汽服门店。
- 比较有商业化气质，适合收费 SaaS。

风险：

- “掌柜”类名称较常见，商标和应用市场重名风险可能较高。

### 备选 2：车修云

推荐理由：

- 简短直接，记忆成本低。
- 与“百易云修”的行业感接近。
- 适合 APP 图标和口头传播。

风险：

- 偏“修”，对洗车、美容店的包容性略弱。

### 备选 3：车服云管家

推荐理由：

- “车服”比“车修”更宽，能覆盖洗美和会员经营。
- “云管家”表达系统属性。

风险：

- “车服”不如“汽服”在行业里自然，但对普通门店老板也容易懂。

### 备选 4：修美云

推荐理由：

- 突出“维修 + 洗美”双业务。
- 名字短，容易做品牌视觉。

风险：

- 听起来更偏洗美，综合汽修厂可能觉得不够专业。

### 备选 5：汽修云管家

推荐理由：

- 直观表达汽修管理系统。
- 容易被搜索和理解。

风险：

- 对洗车店、美容店、综合汽服门店的覆盖感弱于“车店云管家”。

## 3. 我建议采用的品牌结构

主品牌：车店云管家
Web 后台：车店云管家商户后台
安卓 APP：车店云管家商户端
平台后台：车店云管家运营后台

> 注：当前版本不做车主小程序/车主端。车主触达仅保留通知类能力（完工短信、取车提醒、消费提醒）。

官网一句话：

车店云管家是一套面向 1-5 人小型汽修和洗美门店的轻量 SaaS 管理系统，重点解决开单收款、客户车辆、配件库存、质保追溯、会员卡和老板看账。

## 4. 收费模式建议

建议采用「年费套餐 + 增值服务」模式，符合中小汽服门店采购习惯。

### 4.1 套餐年费

单店版：适合 1-5 人单店门店（主力客户）

- 建议价：2980 元/年
- 包含 1 个门店
- 5 个员工账号
- Web 后台 + 安卓 APP
- 客户车辆、预约接待、维修工单、洗美快单
- 储值卡、套餐卡
- 配件库存 + 供应商 + 质保追溯（默认一个仓库，简单入库）
- 老板看账（营业日报、欠款、库存预警）
- A4 工单/结算单
- 基础报表

专业版：适合综合汽服门店

- 建议价：5980 元/年
- 包含 1 个门店
- 15 个员工账号
- 包含单店版全部能力
- 高级库存：盘点、库存预警、成本统计、多仓库
- 技师产值、项目收入、储值/套餐报表
- 更多打印模板
- 数据导出

连锁版：适合多门店商户

- 建议价：12800 元/年起
- 包含 3 个门店
- 50 个员工账号
- 包含专业版全部能力
- 多门店经营看板
- 跨门店客户资产
- 跨门店储值/套餐适用范围
- 总部报表

### 4.2 超额收费

- 增加门店：2000-3000 元/门店/年。
- 增加员工账号：100-200 元/账号/年，或按账号包售卖。
- 短信包：按条或按包收费，后续启用。
- 存储扩容：按容量包收费，适合大量施工照片门店。
- 专属部署/私有化：单独报价，不建议第一阶段主推。

### 4.3 首年销售策略

建议首年降低试用门槛，但不要永久免费：

- 7-15 天免费试用。
- 首年单店版可做 1980 元促销价。
- 老客户续费恢复标准年费。
- 数据初始化、培训、打印模板配置可作为一次性服务费。

### 4.4 不建议的收费方式

- 不建议按工单量收费：门店会担心业务越多越贵。
- 不建议永久免费档：B 端客户支持成本高，免费用户会拖累交付。
- 不建议第一版就按真实支付流水抽佣：当前已经决定不接真实支付，抽佣路径不成立。

## 5. 当前一期商业化边界

第一版收费卖点应聚焦：

- Web 后台 + 安卓 APP。
- 维修工单 + 洗美快单。
- 配件库存 + 供应商 + 质保追溯（小店轻量形态，默认一个仓库）。
- 储值卡 + 套餐卡。
- 老板看账（营业日报、欠款、库存预警）。
- A4 工单和结算单。
- 多商户、多门店、多员工。
- 年费 SaaS，按套餐限制门店数和员工数。

不做车主小程序——通知类能力（完工短信、取车提醒、消费提醒）可保留，但不建设独立车主端。
保险业务、真实支付、复杂营销、工资社保、云仓、EPC 暂不作为第一版卖点。
